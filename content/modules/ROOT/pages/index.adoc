= Advanced Cluster Security Demo 

== Setup Instructions

Run the setup steps in order. Each step depends on the previous ones.

=== Step 1: RHACS Basic Setup

To ensure your RHACS environment is up to date and all necessary components are installed, follow these steps in the *Bastion* tab:

- Ensure you are connected to the Bastion node or terminal provided in your environment.
- This setup process will:
  * Download the latest RHACS demo code and configuration scripts from the official repository.
  * Update your local environment with any new features, components, or bug fixes required for the demo.
  * Install and configure any missing dependencies or prerequisites for the RHACS environment.
  * Set up initial access credentials, such as the RHACS admin password and access URLs.
  * Trigger a compliance scan to ensure the environment is compliant.
- After completing, you will be ready to proceed with additional monitoring and security integrations.

*Procedure*

[start=1]
. Switch to the *Bastion* tab, located on the right side of the screen.
. Run the following
+
[source,sh,subs="attributes",role=execute]
----
cd ~/
git clone https://github.com/mfosterrox/rhacs-demo.git 2>/dev/null || true
cd rhacs-demo && git pull
./basic-setup/install.sh {acs_portal_password}
----

. Verify the deployment completed successfully
+
.**OUTPUT**
[source,sh,subs="attributes"]
----
[INFO] ======================================
[INFO] RHACS Demo Environment Setup Complete!
[INFO] ======================================
[INFO] 
[INFO] RHACS Central Access Information:
[INFO] ==================================
[INFO] Central URL: {acs_route}
[INFO] Username: admin
[INFO] Admin Password: {acs_portal_password}
[INFO] RHACS Version: 4.10.0
[INFO] 
[INFO] 
----

. Source the environment (ROX_CENTRAL_URL, ROX_API_TOKEN) for subsequent steps:
+
[source,sh,subs="attributes",role=execute]
----
source ~/.bashrc
----

=== Step 2: RHACS Monitoring Setup

The `monitoring-setup/install.sh` script sets up secure monitoring integration for RHACS by:
- Deploying a sample Prometheus monitoring stack
- Configuring RHACS authentication/permissions for metrics access
- Verifying the setup and providing troubleshooting guidance if needed
- Integrating with the Perses UI plugin (OpenShift Console) to create a dashboard for monitoring RHACS metrics.


Run the following in the *Bastion* tab. 

*Requirements*

- Requires basic-setup (Step 1) to have been run successfully.

*Procedure*

[start=1]
. Switch to the *Bastion* tab.
. Run the following
+
[source,sh,subs="attributes",role=execute]

cd ~/rhacs-demo/monitoring-setup && ./install.sh

=== Step 3: RHACS VM Vulnerability Scanning

Run the following in the *Bastion* tab to set up RHACS virtual machine vulnerability scanning with OpenShift Virtualization. The script configures RHACS for VM scanning, enables VSOCK, and deploys a RHEL webserver VM with roxagent. When you provide Red Hat subscription credentials, packages install automatically and vulnerability data appears in RHACS within about 10 minutes.

*Requirements*

- Requires basic-setup (Step 1) to have been run successfully.
- *You must have valid Red Hat subscription credentials. (Username/Password)*

[IMPORTANT]
====
You must have valid Red Hat subscription credentials for the VM to be scanned. (Username/Password)
====

*Procedure*

[start=1]
. Switch to the *Bastion* tab.
. Run the following
+
[source,sh,subs="attributes",role=execute]
----
cd ~/rhacs-demo/virt-scanning-setup && ./install.sh
----

. When prompted, enter your Red Hat subscription credentials (username/password). With credentials, the VM registers and installs packages via cloud-init; vulnerability data will appear in RHACS automatically.
. Verify the deployment completed successfully
+
.**OUTPUT**
[source,sh,subs="attributes"]
----
════════════════════════════════════════════════════════════
                    Setup Complete! ✓
════════════════════════════════════════════════════════════

What was configured:

  ✓ virtctl CLI tool installed
  ✓ RHACS Central, Sensor, Collector (ROX_VIRTUAL_MACHINES=true)
  ✓ OpenShift Virtualization VSOCK feature gate enabled
  ✓ Collector hostNetwork=false (pod network)
  ✓ Webserver VM deployed (cloud-init):
    • rhel-webserver

  Now      VM deploying

. (Optional) Verify VM is running

[source,sh,subs="attributes",role=execute]

In the RHACS UI: *Platform Configuration* → *Clusters* → *Virtual Machines*, and *Vulnerability Management* → *Workload CVEs*.

=== Step 4: FIM Demo

Deploys a File Integrity Monitoring policy and triggers violations for demonstration. 

[.requirements]
====
* Requires basic-setup (Step 1) to provide `ROX_API_TOKEN`
* Requires `jq`
====

*Procedure*

[start=1]
. Switch to the *Bastion* tab.
. Ensure `ROX_API_TOKEN` is set (from basic-setup `~/.bashrc`).
. Run the following
+
[source,sh,subs="attributes",role=execute]
----
cd ~/rhacs-demo/fim-setup
git pull && ./install.sh
----

. To trigger FIM violations for demonstration, run the commands printed at the end of the install (or: `oc debug node/<worker>` → `chroot /host` → `touch /etc/passwd`).

[source,sh,subs="attributes",role=execute]
----
oc get nodes -l node-role.kubernetes.io/worker
----

[source,sh,subs="attributes",role=execute]
----
oc debug node/<worker-node-name>
----

[source,sh,subs="attributes",role=execute]
----
chroot /host
touch /etc/passwd
----

. Type "exit" twice to exit the pod
