= Advanced Cluster Security Demo 

== Setup Instructions

Below are the instrcutions to set up the demo. 

There is a single installation script called `install-all-setup.sh`. It initializes are runs 6 seperate scripts.

First, it runs the *basic-setup* then runs the other 5 scripts in parallel including:

* FAM (file activity monitoring), 
* Persus monitoring in the console, 
* MCP server setup
* Vulnerability management for Virtual Machines
* Policy as Code (GitOps) via Argo CD

=== Requirements

All requirements are setup in the *basic-setup* script. However, if you want to set up the VM scanning stack, you will need to create an activation key for RHEL 9 (see <<activation-key-rhel9,activation key instructions>>).

You can also message @foster on slackfor an organization ID and activation key, but he would prefer you create your own.

NOTE: If you do not require the VM scanning stack, you can skip the activation key creation step. The installation script will skip the VM scanning set up.

[[activation-key-rhel9]]
=== Creating an activation key for RHEL 9

All Red Hatters should have access to the https://console.redhat.com[Red Hat Hybrid Cloud Console,window=blank] to create an activation key.

.Procedure
. Log in to the https://console.redhat.com[Red Hat Hybrid Cloud Console,window=blank].
. Navigate to *Services* > *System Configuration* > *Activation Keys* (or, in some views, *Inventory* > *System Configuration* > *Activation Keys*).
. Click *Create activation key* and provide:
+
--

* Name and Description
** Name — Use the default
** Description (optional)

* *Workload*
** Latest Release

* *System purpose* - Development/Test
** Role: Red Hat Linux Server
** Service Level Agreement (SLA): Self-Support
** Usage: Development/Test

* *Service Level Agreement (SLA)* — Self-Support
* *System purpose* - Development/Test
* *Subscriptions* — Select the Red Hat subscriptions or pools you want this key to auto-attach (for example, RHEL entitlements for physical or virtual systems). You can enable auto-attach if desired.
--
. Review and click *Create*.

IMPORTANT: Note the following for use with `install-all-setup.sh`:

* The activation key *name*.
* Your organization's *Organization ID* (a numeric value, visible at the top of the Activation Keys page or under your account details).

=== Set up scripts

.Procedure
. Copy and paste the following command into your terminal:
+
[source,sh,subs="attributes",role=execute]
----
source ~/.bashrc
cd ~/
git clone https://github.com/mfosterrox/rhacs-demo.git 2>/dev/null || true
cd ~/rhacs-demo && git pull
chmod +x install-all-setup.sh verify-all-setup.sh
./install-all-setup.sh -p {acs_portal_password}
----

. Follow the prompts:
+
* *Red Hat subscription* — enter organization ID and activation key for VM scanning, or press Enter at the organization prompt to skip `virt-scanning-setup`.

. Wait until the script finishes. You should see *Phase 1* (basic-setup) complete, then *Phase 2* parallel jobs start and *Waiting for N parallel job(s)* complete without errors.

IMPORTANT: If a script fails, you can run the indivdual script again. The error messages will provide the command to rerun the script.

=== Verify success

Run the verification script.

[source,sh,subs="attributes",role=execute]
----
cd ~/rhacs-demo
source ~/.bashrc
./verify-all-setup.sh
----

OUTPUT:
[source,text]
----
[STEP] Summary
  ✓ No failed checks (0 warning(s))
----
