= Advanced Cluster Security Demo 

== Setup Instructions

Use the single orchestration script `install-all-setup.sh` at the root of the demo repository. It runs *basic-setup* first (so RHACS Central stays stable while APIs are configured), then runs the other demo stacks in parallel: Lightspeed, FIM, monitoring, MCP, and VM vulnerability scanning.

=== Requirements

Before you start, on the *Bastion* host collect the following information if you want to use the optional features:

* *Optional — Red Hat subscription* — username and password to entitle the demo Virtual Machine (`virt-scanning-setup`).
* *Optional — OpenShift Lightspeed OLSConfig* — an OpenAI-compatible API key (and optional model/URL) if you want the "Ask OpenShift Lightspeed" flow.

=== Procedure

[start=1]
. Open the *Bastion* tab.
. Clone or update the repository and run the installer:
+
[source,sh,subs="attributes",role=execute]
----
cd ~/
git clone https://github.com/mfosterrox/rhacs-demo.git 2>/dev/null || true
cd ~/rhacs-demo && git pull
chmod +x install-all-setup.sh verify-all-setup.sh
./install-all-setup.sh -p {acs_portal_password}
----

. Follow the prompts:
+
* *Red Hat subscription* — enter credentials for VM scanning, or press Enter to skip `virt-scanning-setup`.
* *Lightspeed OLS* — answer `y` and provide an API key (and defaults for model/URL if prompted), or `N` to skip creating `OLSConfig` while still installing the operator.

. Wait until the script finishes. You should see *Phase 1* (basic-setup) complete, then *Phase 2* parallel jobs start and *Waiting for N parallel job(s)* complete without errors.

. Source your shell profile so `ROX_CENTRAL_URL`, `ROX_API_TOKEN`, and related variables are available in new shells:
+
[source,sh,subs="attributes",role=execute]
----
source ~/.bashrc
----

=== Verify success

Run the verification script from the repo root. It checks cluster state and (when `ROX_API_TOKEN` is set) RHACS API expectations for each stack:

[source,sh,subs="attributes",role=execute]
----
cd ~/rhacs-demo
source ~/.bashrc
./verify-all-setup.sh
----

* Exit code *0* — no failed checks (warnings may appear for optional items such as OLS not configured).
* Exit code *1* — at least one check failed; inspect the printed messages.

=== Expected success output (install)

When `install-all-setup.sh` completes, you should see *All setup phases completed successfully.* RHACS Central access matches your environment, for example:

[source,sh,subs="attributes"]
----
[INFO] Central URL: {acs_route}
[INFO] Username: admin
[INFO] Admin Password: {acs_portal_password}
----

Those values are also shown at the end of `basic-setup` in `basic-setup.log` if you need to confirm.

=== FIM demo (after install)

To trigger File Integrity Monitoring violations for demonstration, use the worker node commands printed at the end of `fim-setup` in the parallel logs, or:

. Get the first worker node and start a debug session:
+
[source,sh,subs="attributes",role=execute]
----
worker_nodes=$(oc get nodes -l node-role.kubernetes.io/worker --no-headers -o custom-columns=":metadata.name")
oc debug node/$(echo "$worker_nodes" | head -n1)
----

. Inside the debug shell, chroot to the host:
+
[source,sh]
----
chroot /host
----

. Trigger a monitored path change (simulating a FIM violation):
+
[source,sh]
----
touch /etc/passwd
----

. Type `exit` twice to leave the debug shell (once for chroot, once to exit the debug pod).
